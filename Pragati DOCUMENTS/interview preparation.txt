1. What is HTML?Answer: HTML (HyperText Markup Language) is the standard language used to create the structure of web pages using elements called tags.2. What are semantic tags in HTML?Answer: Semantic tags clearly describe the meaning of the content they hold. Examples: <header>, <footer>, <section>, <article>, <nav>.3. What is the difference between <div> and <span>?Answer:<div> is a block-level element (takes the whole width).<span> is an inline element (only takes the space it needs).4. What does <!DOCTYPE html> mean?Answer: It tells the browser that the page is written in HTML5.5. What is the difference between HTML and HTML5?Answer:HTML5 is the latest version with support for audio, video, semantic elements, and better error handling.?? CSS Interview Questions with Answers1. What is CSS?Answer: CSS (Cascading Style Sheets) is used to style HTML content—like colors, spacing, fonts, and layout.2. Difference between id and class in CSS?Answer:id is unique for one element (#id).class can be reused on many elements (.class).3. What is the Box Model in CSS?Answer:The box model includes:Content ? Padding ? Border ? Margin, which defines how an element is spaced and sized.4. What are pseudo-classes in CSS?Answer:Pseudo-classes define special states of elements. Examples::hover – when the mouse is over an element:first-child – first child of a parent5. What is the difference between inline, block, and inline-block elements?Answer:inline: Does not start on a new line (<span>)block: Starts on a new line (<div>)inline-block: Like inline, but allows setting width/height?? JavaScript Interview Questions with Answers1. What is JavaScript?Answer: JavaScript is a scripting language used to make web pages interactive (like buttons, forms, sliders).2. Difference between var, let, and const?var: Function-scoped, can be redeclared.let: Block-scoped, can't be redeclared.const: Block-scoped, cannot be changed after assignment.3. What is hoisting?Answer: JavaScript moves variable and function declarations to the top of their scope before execution.4. What is a closure?Answer: A closure is a function that remembers variables from its outer function even after the outer function has finished running.5. Difference between == and ===?==: Compares values only (loose equality).===: Compares value and type (strict equality).6. What is a promise in JavaScript?Answer: A Promise represents a task that will finish in the future, either successfully (resolve) or with an error (reject).7. What is an arrow function?Answer: A shorter way to write a function. It doesn’t have its own this.Example: const sum = (a, b) => a + b;8. What is the event loop?Answer: The event loop lets JavaScript handle asynchronous tasks by putting them in a queue and executing them one by one when the main code is done.9. What is this in JavaScript?Answer: this refers to the object that is currently executing the function.?? React JS Interview Questions with Answers1. What is React JS?Answer: React is a JavaScript library used to build fast and interactive user interfaces, especially single-page apps.2. What is a component in React?Answer: Components are reusable pieces of UI. They can be functional or class-based.3. What is JSX?Answer: JSX is a syntax extension that lets you write HTML-like code in JavaScript.Example: <h1>Hello</h1>4. What are props in React?Answer: Props (short for properties) are used to pass data from parent to child components.5. What is state in React?Answer: State is used to store data inside a component and update the UI when the data changes.6. Difference between props and state?Props: Passed from outside, read-only.State: Managed inside the component, can be changed.7. What are React hooks?Answer: Hooks are special functions (like useState, useEffect) that let you use features like state and lifecycle in functional components.8. What is useEffect used for?Answer: useEffect is used to perform side effects like fetching data, setting up timers, or listening for changes.
